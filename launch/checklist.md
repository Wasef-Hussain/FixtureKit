# 48-Hour Launch Checklist

## Before you post (pre-launch)

- [ ] Push final build to hosting (Vercel / Netlify / Cloudflare Pages)
- [ ] Verify live URL loads and works end-to-end (TS mode + Zod mode)
- [ ] Add the live URL to all post drafts (replace `[link]` placeholders)
- [ ] Push `README.md` to GitHub
- [ ] Set GitHub repo description: *"Paste a TypeScript interface or Zod schema → get fixture code"*
- [ ] Add `typescript`, `zod`, `testing`, `fixtures`, `developer-tools` topics to the GitHub repo
- [ ] Take 3 screenshots matching the captions in `launch/captions.md`
- [ ] Upload screenshots to GitHub repo `/docs` or `/screenshots` folder

---

## Hour 0 — Post

- [ ] Submit **Show HN** on Hacker News (aim for 9–11 AM EST weekday for best traffic)
- [ ] Post on **r/typescript** with the Reddit draft
- [ ] Post on **DEV.to** with the DEV.to draft (schedule or publish immediately)
- [ ] Cross-post to **r/webdev** (same Reddit draft, check subreddit rules)

---

## Hour 0–6 — Engage

- [ ] Reply to every HN comment within the first 2 hours
- [ ] Reply to Reddit comments; thank people who try it and report bugs
- [ ] Fix any critical bugs surfaced by early users and redeploy
- [ ] Note any schema patterns that break the parser (future scope)

---

## Hour 6–24 — Amplify

- [ ] Share on Twitter/X: one short post with a GIF or screenshot + link
- [ ] Post in any TypeScript/testing Discord servers you're active in
- [ ] Submit to any relevant newsletters or link aggregators you know (e.g. TypeScript Weekly, This Week in React if relevant)

---

## Hour 24–48 — Follow up

- [ ] Reply to any remaining unanswered comments on all platforms
- [ ] Post a brief "what I learned" thread on Twitter/X or DEV.to if HN discussion was substantial
- [ ] Check analytics (if set up): what pages or referrers are driving traffic
- [ ] Open GitHub issues for the top requested features or reported bugs
- [ ] Decide on next version scope based on feedback
